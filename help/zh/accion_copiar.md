# 操作：复制

目标：从源端发送快照，在目标端接收。

条件：

- 源：已选择快照。
- 目标：已选择数据集。
- 源和目标都必须使用 OpenZFS `2.3.3` 或更高版本。

行为：

- 使用 `zfs send` 与 `zfs recv`。
- 当两端连接都有支持 jobs 的活跃 daemon（`JOBS_SUPPORT=1`）时，传输以**后台 job** 方式运行：
  - 数据直接在两个 daemon 之间流动，不经过运行 ZFSMgr 的机器。
  - 界面不会阻塞；job 立即启动，应用仍可正常使用。
  - 进度显示在 **Transferencias** 标签页（字节数、速率、耗时）。
  - 传输进行中可以关闭 GUI，daemon 会继续执行。
  - 重新打开 GUI 或重连后，正在运行的 job 会自动恢复。
  - 可在 Transferencias 标签页取消任意 job（向 `zfs send` 进程发送 `SIGTERM`）。
- 如果任一 daemon 不支持 jobs，则回退到同步模式：加入 `待执行更改`，在应用更改时执行。
- 完成后刷新受影响的连接/树。
- 若任一端低于 `2.3.3`，该操作会被阻止。
